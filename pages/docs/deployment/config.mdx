# Config

This document describes the `statickit.json` schema. These are the available top-level keys (none are required):

| Key                     | Type                     | Description                                                  |
| :---------------------- | :----------------------- | :----------------------------------------------------------- |
| [`forms`](#forms)       | Object                   | An object describing forms to provision                      |

## `forms`

An object containing forms to provision. Upon deployment, if a form does not exist for a particular key, we'll create one. If the form already exists, we'll update its settings to match what's provided.

The key can be used to identify the form when wiring it up on the frontend:

```js
// statickit.json
{
  "forms": {
    "newsletter": {
      "name": "Newsletter Opt-In"
    }
  }
}

// In your JavaScript
sk('site', 'init', {
  key: 'YOUR_SITE_PUBLIC_KEY'
})

sk('form', 'init', {
  key: 'newsletter',
  element: '#my-form'
})
```

Values in the `forms` object can contain the following fields (all are optional unless marked as required):

| Key                     | Type                     | Description                                                  |
| :---------------------- | :----------------------- | :----------------------------------------------------------- |
| `name`                  | String                   | The name of the form (required)                              |
| `actions`               | Object                   | An object containing actions to perform                      |
| `fields`                | Object                   | An object containing field configuration                     |
